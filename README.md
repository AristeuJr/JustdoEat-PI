# JustdoEat-PI

Com certeza! O projeto "Just do Eat" possui uma proposta social muito interessante e um escopo bem estruturado. Com base no documento fornecido, preparei um resumo organizado para o seu README:

### Objetivo Geral

* [cite_start]O objetivo geral do projeto "Just do Eat" é desenvolver uma plataforma web intuitiva de delivery que conecta usuários a restaurantes e, de forma simultânea, auxilia instituições carentes[cite: 18]. 
* [cite_start]A solução busca unir a praticidade e a conveniência do pedido de refeições online com a responsabilidade social, gerando doações automaticamente através das compras realizadas no sistema por meio de um programa chamado "Parcela Amiga"[cite: 18, 60, 61].

### Objetivos Específicos

* [cite_start]Simplificar o processo de compra e entrega de produtos alimentícios, promovendo acessibilidade digital para diferentes perfis de usuários[cite: 21].
* [cite_start]Ampliar o alcance de ações solidárias no cotidiano das pessoas, resolvendo a dificuldade que instituições carentes enfrentam para obter alimentos fora de datas comemorativas[cite: 21, 65, 66].
* [cite_start]Destinar, através do mecanismo "Parcela Amiga", uma porcentagem dos lucros de cada transação para um fundo solidário que é revertido em alimentos e suprimentos básicos para as instituições parceiras cadastradas[cite: 87, 88].
* [cite_start]Contribuir ativamente para os Objetivos de Desenvolvimento Sustentável (ODS) da ONU, com foco no ODS 2 (Fome Zero e Agricultura Sustentável), ODS 3 (Boa Saúde e Bem-Estar) e ODS 9 (Indústria, inovação e infraestrutura)[cite: 21, 88, 140].
* [cite_start]Fornecer um ecossistema completo que inclui a gestão de perfis (clientes, restaurantes e administradores), sistema de carrinho e checkout, rastreamento de entregas em tempo real e um sistema de avaliação de serviços[cite: 83, 118, 120, 121, 123].

### Descrição das Interfaces já desenvolvidas

* [cite_start]**Homepage (Layout Web):** A tela inicial conta com uma interface amigável que apresenta uma barra de pesquisa, divisão rápida por categorias (como Lanches, Marmitas e Bebidas), uma vitrine destacando os melhores restaurantes e uma área de chamada (Call to Action) para engajar o usuário a contribuir com a causa social[cite: 140].
* [cite_start]**Suporte / FAQ:** Uma página focada no atendimento ao usuário, contendo uma barra de pesquisa de ajuda, menus expansíveis para dúvidas frequentes (como etapas para ser parceiro ou cadastrar estabelecimento) e um espaço para envio de feedback para a melhoria do software[cite: 140].
* [cite_start]**Quem Somos Nós:** Uma tela institucional que exibe claramente a missão do projeto e detalha visualmente os Objetivos de Desenvolvimento Sustentável (ODS) da ONU que a plataforma atende[cite: 140].
* **Login e Registro (Cadastro):** Telas limpas para autenticação de usuários. [cite_start]A interface de registro é abrangente e se adapta ao tipo de usuário (Cliente ou Restaurante), capturando dados essenciais como Nome/Razão Social, CPF/CNPJ, Endereço completo, Telefone e Horários de Abertura/Fechamento[cite: 140].
* **Dashboard Administrativo:** Um painel de controle focado na gestão do negócio (voltado para restaurantes/administração). [cite_start]A aba "Visão Geral" exibe métricas vitais como número de pedidos ativos, valor das vendas do dia, ranking de itens mais vendidos e histórico de avaliações recentes[cite: 140].
* **Listar (Cadastro / Edição de Produtos):** Uma interface dupla que funciona como gestor de cardápio. Do lado esquerdo, lista os produtos já disponíveis com foto, nome, descrição, preço e status (ativo). [cite_start]Do lado direito, apresenta um formulário para cadastrar novos produtos ou editar os existentes, definindo categorias e opções adicionais[cite: 140].
